## ✨ 功能特色

### 🎯 核心搶票功能
| 功能 | 說明 |
|------|------|
| **自動點擊購票按鈕** | 偵測到「立即購票」按鈕後光速點擊 |
| **自動選擇區域** | 根據票價/區域關鍵字自動選擇座位區 |
| **自動填寫表單** | 自動選張數、勾同意條款、聚焦驗證碼框 |
| **Enter 自動提交** | 輸入完驗證碼按 Enter 直接送出，省去找按鈕的時間 |
| **多組關鍵字備選** | 支援逗號分隔多組關鍵字（如 `5800, 4800, 3800`），依序嘗試 |

### 🚀 速度優化
| 功能 | 說明 |
|------|------|
| **移除網頁動畫** | 拔除所有 CSS transition/animation，頁面瞬間載入 |
| **開賣前自動重整** | 尚未開賣時自動刷新頁面，開賣瞬間搶先進入 |
| **智慧重整間隔** | 可選 500ms ~ 2000ms 間隔，自動加入隨機抖動避免被偵測 |
| **攔截彈出視窗** | 自動攔截 `alert` / `confirm` 彈窗，不會被「請勿重新整理」卡住 |
| **驗證碼圖片放大** | 自動放大 1.5 倍並增強清晰度，看得更快更準 |

### 💎 使用者體驗
| 功能 | 說明 |
|------|------|
| **即時狀態指示燈** | 🟢 運作中 / 🟡 待命 / 🔴 非拓元頁面，一目瞭然 |
| **操作日誌面板** | 可收合的即時日誌，不用開 DevTools 就能看進度 |
| **倒數計時器** | 設定開賣時間，即時倒數，5 分鐘內進入緊張模式 |
| **設定匯出/匯入** | 一鍵備份設定為 JSON 檔案，換電腦也不怕 |

## 📦 安裝方式

### 方法一：手動安裝（開發者模式）

1. **下載專案**
   ```
   git clone https://github.com/你的帳號/tixcraft-helper.git
   ```
   或直接下載 ZIP 解壓縮

2. **開啟 Chrome 擴充功能頁面**
   - 在網址列輸入 `chrome://extensions/`
   - 或從選單 → 更多工具 → 擴充功能

3. **啟用開發者模式**
   - 右上角開啟「開發人員模式」開關

4. **載入擴充功能**
   - 點擊「載入未封裝項目」
   - 選擇本專案的資料夾

5. **完成！** 你會在工具列看到擴充功能圖示

## 🎮 使用教學

### 基本設定

1. 點擊工具列上的擴充功能圖示，打開設定面板
2. 勾選 **「啟用輔助功能」**
3. 依需求填寫以下設定：

| 設定 | 範例 | 說明 |
|------|------|------|
| 場次/日期關鍵字 | `12/31, 1/1` | 逗號分隔多組，依序嘗試。留空則自動選第一個 |
| 票價/區域關鍵字 | `5800, 4800, 3800` | 逗號分隔多組，搶不到第一志願自動切換 |
| 購買張數 | `2` | 選 1~4 張 |

4. 點擊 **「儲存設定」**

### 進階設定

- **🚀 移除網頁過場動畫** — 建議開啟，頁面載入更快
- **🔄 開賣前自動重整** — 搶票日當天開啟，選擇適合的重整間隔
- **⏰ 開賣時間** — 設定後會顯示倒數計時

### 搶票流程（全自動）

```
開啟活動頁面 → [自動點擊購票] → [自動選區域] → [自動填張數+勾同意]
                                                        ↓
                                              🎯 驗證碼框自動聚焦
                                                        ↓
                                              ⌨️ 輸入驗證碼 + Enter
                                                        ↓
                                              🚀 自動提交！搶票完成！
```

> 💡 **小技巧：** 同時開 2~3 個分頁，每個分頁會獨立運作，等於同時搶多次！

## 🔧 技術細節

### 架構設計

- **Manifest V3** — 遵循最新的 Chrome 擴充功能標準
- **Content Script** — 注入到 tixcraft.com 頁面，自動化操作 DOM
- **MAIN World Script** — `anti_alert.js` 在頁面主環境執行，繞過 CSP 限制
- **Chrome Storage API** — 儲存使用者設定，跨分頁共享
- **Message Passing** — Popup 與 Content Script 即時通訊

### 核心技術

| 技術 | 用途 |
|------|------|
| `requestAnimationFrame` | 零延遲輪詢，比 `setInterval` 更快 |
| `chrome.runtime.onMessage` | Popup ↔ Content Script 雙向通訊 |
| `world: "MAIN"` | 在頁面主執行環境中攔截 alert/confirm |
| Random Jitter | 重整間隔加入 ±150ms 隨機抖動，降低被偵測風險 |
| `dispatchEvent` | 觸發原生事件，確保前端框架正確接收狀態變更 |

### 支援的頁面路徑

| 路徑 | 功能 |
|------|------|
| `/activity/detail/*` | 自動點擊購票按鈕 |
| `/activity/game/*` | 自動點擊購票按鈕（場次列表） |
| `/ticket/area/*` | 自動選擇區域 |
| `/ticket/ticket/*` | 自動填表 + 驗證碼聚焦 |

## ⚠️ 注意事項

> [!IMPORTANT]
> - 本工具僅作為**輔助**用途，不保證一定能搶到票
> - 驗證碼仍需**手動輸入**，這是最後一道防線
> - 請遵守拓元售票系統的使用條款
> - 過於頻繁的自動重整可能導致 IP 被暫時封鎖，建議使用 800ms 以上的間隔

> [!WARNING]
> - 不要同時開啟過多分頁（建議 2~3 個即可）
> - 搶票前請先測試設定是否正確
> - 建議在非搶票時段先熟悉操作流程
